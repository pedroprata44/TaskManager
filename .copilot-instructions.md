# Copilot Instructions

## Project Structure

- Project layout: `.NET` minimal API in `TaskManager.Api` and xUnit tests in `TaskManager.Tests`.

## Domain Models

- Domain models in `TaskManager.Api/Models` must include audit timestamp fields: `CreatedAt` and `UpdatedAt`.
- Required properties should be explicit. For tasks, `Title` and `Description` are required.
- Initialize `string` properties with `string.Empty` and default boolean properties to `false`.
- Use `Guid` identifiers and create new entities with `Guid.NewGuid()`.

## Architecture

- Prefer interface segregation; depend on service interfaces like `ITaskService`, not concrete implementations.

## Testing

- In xUnit tests:
  - use randomized domain data instead of hardcoded values
  - use the test class constructor for setup
  - implement `IDisposable` for cleanup when necessary
- Run tests after every project change and ensure the test suite passes.

## Behavior

- For missing resources, operations such as `GetById`, `Update`, and `Delete` must throw exceptions (for example, `KeyNotFoundException`) rather than return `null`.
- Follow TDD when feasible: write tests before implementing new behavior.
- Keep the API minimal and do not add example endpoints unless requested.
