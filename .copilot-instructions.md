# Copilot Instructions

- This repository contains a .NET minimal API project in `TaskManager.Api` and an xUnit project in `TaskManager.Tests`.
- All domain models in `TaskManager.Api/Models` should include audit timestamp fields: `CreatedAt` and `UpdatedAt`.
- Models should express required data explicitly when applicable. For example, `Task` requires `Title` and `Description`.
- When using `string` properties, initialize them with `string.Empty` by default.
- Boolean model properties should default to `false` when appropriate.
- Identifiers should use `Guid` and start with `Guid.NewGuid()` for new entities.
- Use randomized test data in xUnit tests when creating domain objects so repeated runs do not depend on hardcoded values.
- Use interface segregation and depend on service interfaces like `ITaskService` rather than concrete implementations.
- In xUnit tests, use the test class constructor for setup and implement `IDisposable` for cleanup when necessary.
- Run tests after every change in files and ensure every test passes.
- When making any project change, run the test project and ensure every test passes.
- Follow Test-Driven Development (TDD) by writing tests first before implementing new behavior whenever feasible.
- Keep the API minimal and avoid adding example endpoints unless requested.
